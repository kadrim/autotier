◆ setFallocateWithKeepSize()

EnvOptions org.rocksdb.EnvOptions.setFallocateWithKeepSize ( final boolean  fallocateWithKeepSize)
inline

If true, we will preallocate the file with

FALLOC_FL_KEEP_SIZE

flag, which means that file size won't change as part of preallocation. If false, preallocation will also change the file size. This option will improve the performance in workloads where you sync the data on every write. By default, we set it to true for MANIFEST writes and false for WAL writes
