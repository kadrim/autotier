581  // For a given key, check to see if there are any records for this key
582  // in the memtables, including memtable history. If cache_only is false,
583  // SST files will also be checked.
584  //
585  // If a key is found, *found_record_for_key will be set to true and
586  // *seq will be set to the stored sequence number for the latest
587  // operation on this key or kMaxSequenceNumber if unknown.
588  // If no key is found, *found_record_for_key will be set to false.
589  //
590  // Note: If cache_only=false, it is possible for *seq to be set to 0 if
591  // the sequence number has been cleared from the record. If the caller is
592  // holding an active db snapshot, we know the missing sequence must be less
593  // than the snapshot's sequence number (sequence numbers are only cleared
594  // when there are no earlier active snapshots).
595  //
596  // If NotFound is returned and found_record_for_key is set to false, then no
597  // record for this key was found. If the caller is holding an active db
598  // snapshot, we know that no key could have existing after this snapshot
599  // (since we do not compact keys that have an earlier snapshot).
600  //
601  // Only records newer than or at `lower_bound_seq` are guaranteed to be
602  // returned. Memtables and files may not be checked if it only contains data
603  // older than `lower_bound_seq`.
604  //
605  // Returns OK or NotFound on success,
