◆ getExpiration()

long org.rocksdb.TransactionOptions.getExpiration ( )
inline

Expiration duration in milliseconds.

If non-negative, transactions that last longer than this many milliseconds will fail to commit. If not set, a forgotten transaction that is never committed, rolled back, or deleted will never relinquish any locks it holds. This could prevent keys from being written by other writers.

Returns
expiration the expiration duration in milliseconds

boolean org.rocksdb.TransactionOptions.isDeadlockDetect ( )

True means that before acquiring locks, this transaction will check if doing so will cause a deadlock. If so, it will return with Status.Code#Busy. The user should retry their transaction.
