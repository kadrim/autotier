Detailed Description

SstFileManager is used to track SST files in the DB and control their deletion rate.

All SstFileManager public functions are thread-safe.

SstFileManager is not extensible.

== Deletion rate limiting specific arguments ==

Parameters
rateBytesPerSechow many bytes should be deleted per second, If this value is set to 1024 (1 Kb / sec) and we deleted a file of size 4 Kb in 1 second, we will wait for another 3 seconds before we delete other files, Set to 0 to disable deletion rate limiting.
maxTrashDbRatioif the trash size constitutes for more than this fraction of the total DB size we will start deleting new files passed to DeleteScheduler immediately.
bytesMaxDeleteChunkif a single file is larger than delete chunk, ftruncate the file by this size each time, rather than dropping the whole file. 0 means to always delete the whole file.
