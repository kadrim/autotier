14 // Computes a stable, universally unique 192-bit (24 binary char) identifier
15 // for an SST file from TableProperties. This is supported for table (SST)
16 // files created with RocksDB 6.24 and later. NotSupported will be returned
17 // for other cases. The first 16 bytes (128 bits) is of sufficient quality
18 // for almost all applications, and shorter prefixes are usable as a
19 // hash of the full unique id.
20 //
21 // Note: .c_str() is not compatible with binary char strings, so using
22 // .c_str() on the result will often result in information loss and very
23 // poor uniqueness probability.
24 //
25 // More detail: the first 128 bits are *guaranteed* unique for SST files
26 // generated in the same process (even different DBs, RocksDB >= 6.26),
27 // and first 128 bits are guaranteed not "all zeros" (RocksDB >= 6.26)
28 // so that the "all zeros" value can be used reliably for a null ID.
29 // Assuming one generates many SST files in the lifetime of each process,
30 // the probability of collision between processes is "better than
31 // random": if processes generate n SST files on average, we expect to
32 // generate roughly 2^64 * sqrt(n) files before first collision in the
33 // first 128 bits. See https://github.com/pdillinger/unique_id
34 // Using the full 192 bits, we expect to generate roughly 2^96 * sqrt(n)
35 // files before first collision.
36 Status GetUniqueIdFromTableProperties(const TableProperties &props,
37  std::string *out_id);
