◆ LockFile()

Status ROCKSDB_NAMESPACE::EnvLibrados::LockFile ( const std::string &  fname,
FileLock **  lock 
)
overridevirtual

lock file. create if missing.

[long description]

It seems that LockFile is used for preventing other instance of RocksDB from opening up the database at the same time. From RocksDB source code, the invokes of LockFile are at following locations:

./db/db_impl.cc:1159: s = env_->LockFile(LockFileName(dbname_), &db_lock_); // DBImpl::Recover ./db/db_impl.cc:5839: Status result = env->LockFile(lockname, &lock); // Status DestroyDB

When db recovery and db destroy, RocksDB will call LockFile
