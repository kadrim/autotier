Detailed Description

Offers functionality for implementations of AbstractNativeReference which have an immutable reference to the underlying native C++ object

Member Function Documentation

◆ close()

void org.rocksdb.AbstractImmutableNativeReference.close ( )
inline

Frees the underlying C++ object

It is strong recommended that the developer calls this after they have finished using the object.

Note, that once an instance of AbstractNativeReference has been disposed, calling any of its functions will lead to undefined behavior.

◆ disOwnNativeHandle()

final void org.rocksdb.AbstractImmutableNativeReference.disOwnNativeHandle ( )
inlineprotected

Releases this

AbstractNativeReference

from the responsibility of freeing the underlying native C++ object

This will prevent the object from attempting to delete the underlying native object in its finalizer. This must be used when another object takes over ownership of the native object or both will attempt to delete the underlying object when garbage collected.

When

disOwnNativeHandle()

is called,

dispose()

will subsequently take no action. As a result, incorrect use of this function may cause a memory leak.

See also
dispose()
