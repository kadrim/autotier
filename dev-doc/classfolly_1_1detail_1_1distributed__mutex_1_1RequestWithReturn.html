Detailed Description

template<typename Func>
class folly::detail::distributed_mutex::RequestWithReturn< Func >

A template that helps us differentiate between the different ways to return a value from a combined critical section. A return value of type void cannot be stored anywhere, so we use specializations and pick the right one switched through std::conditional_t

This is then used by CoalescedTask and its family of functions to implement efficient return value transfers to the waiting threads

We need to define the destructor here because C++ requires (with good reason) that a union with non-default destructor be explicitly destroyed from the surrounding class, as neither the runtime nor compiler have the knowledge of what to do with a union at the time of destruction

Each request that has a valid return value set will have the value retrieved from the get() method, where the value is destroyed. So we don't need to destroy it here
