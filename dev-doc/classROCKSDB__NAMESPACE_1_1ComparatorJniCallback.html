Detailed Description

This class acts as a bridge between C++ and Java. The methods in this class will be called back from the RocksDB storage engine (C++) we then callback to the appropriate Java method this enables Comparators to be implemented in Java.

The design of this Comparator caches the Java Slice objects that are used in the compare and findShortestSeparator method callbacks. Instead of creating new objects for each callback of those functions, by reuse via setHandle we are a lot faster; Unfortunately this means that we have to introduce independent locking in regions of each of those methods via the mutexs mtx_compare and mtx_findShortestSeparator respectively
