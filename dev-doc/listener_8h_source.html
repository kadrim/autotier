452 // EventListener class contains a set of callback functions that will
453 // be called when specific RocksDB event happens such as flush. It can
454 // be used as a building block for developing custom features such as
455 // stats-collector or external compaction algorithm.
456 //
457 // IMPORTANT
458 // Because compaction is needed to resolve a "writes stopped" condition,
459 // calling or waiting for any blocking DB write function (no_slowdown=false)
460 // from a compaction-related listener callback can hang RocksDB. For DB
461 // writes from a callback we recommend a WriteBatch and no_slowdown=true,
462 // because the WriteBatch can accumulate writes for later in case DB::Write
463 // returns Status::Incomplete. Similarly, calling CompactRange or similar
464 // could hang by waiting for a background worker that is occupied until the
465 // callback returns.
466 //
467 // Otherwise, callback functions should not run for an extended period of
468 // time before the function returns, because this will slow RocksDB.
469 //
470 // [Threading] All EventListener callback will be called using the
471 // actual thread that involves in that specific event. For example, it
472 // is the RocksDB background flush thread that does the actual flush to
473 // call EventListener::OnFlushCompleted().
474 //
475 // [Locking] All EventListener callbacks are designed to be called without
476 // the current thread holding any DB mutex. This is to prevent potential
477 // deadlock and performance issue when using EventListener callback
478 // in a complex way.

480  public:
481  static const char* Type() { return "EventListener"; }
482  static Status CreateFromString(const ConfigOptions& options,
483  const std::string& id,
484  std::shared_ptr<EventListener>* result);
485  const char* Name() const override {
486  // Since EventListeners did not have a name previously, we will assume
487  // an empty name. Instances should override this method.
488  return "";
489  }
490  // A callback function to RocksDB which will be called whenever a
491  // registered RocksDB flushes a file. The default implementation is
492  // no-op.
493  //
494  // Note that the this function must be implemented in a way such that
495  // it should not run for an extended period of time before the function
496  // returns. Otherwise, RocksDB may be blocked.
497  virtual void OnFlushCompleted(DB* /*db*/,
498  const FlushJobInfo& /*flush_job_info*/) {}
