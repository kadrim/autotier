CompactionOptions org.rocksdb.CompactionOptions.setCompression ( final CompressionType  compression)
inline

Set the compaction output compression type.

Default: snappy

If set to CompressionType#DISABLE_COMPRESSION_OPTION, RocksDB will choose compression type according to the ColumnFamilyOptions#compressionType(), taking into account the output level if ColumnFamilyOptions#compressionPerLevel() is specified.

◆ setMaxSubcompactions()

CompactionOptions org.rocksdb.CompactionOptions.setMaxSubcompactions ( final int  maxSubcompactions)
inline

This value represents the maximum number of threads that will concurrently perform a compaction job by breaking it into multiple, smaller ones that are run simultaneously.

Default: 0 (i.e. no subcompactions)

If > 0, it will replace the option in DBOptions#maxSubcompactions() for this compaction.

Parameters
maxSubcompactionsThe maximum number of threads that will concurrently perform a compaction job

the instance of the current Options.
