Detailed Description

ColumnFamilyHandle class to hold handles to underlying rocksdb ColumnFamily Pointers.

Member Function Documentation

◆ disposeInternal()

void org.rocksdb.ColumnFamilyHandle.disposeInternal ( )
inlineprotected

Deletes underlying C++ iterator pointer.

Note: the underlying handle can only be safely deleted if the RocksDB instance related to a certain ColumnFamilyHandle is still valid and initialized. Therefore

disposeInternal()

checks if the RocksDB is initialized before freeing the native handle.

Reimplemented from org.rocksdb.RocksObject.

◆ getDescriptor()

ColumnFamilyDescriptor org.rocksdb.ColumnFamilyHandle.getDescriptor ( ) throws RocksDBException
inline

Gets the up-to-date descriptor of the column family associated with this handle. Since it fills "*desc" with the up-to-date information, this call might internally lock and release DB mutex to access the up-to-date CF options. In addition, all the pointer-typed options cannot be referenced any longer than the original options exist.

Note that this function is not supported in RocksDBLite.

Returns
the up-to-date descriptor.
Exceptions
RocksDBExceptionif an error occurs whilst retrieving the descriptor.
